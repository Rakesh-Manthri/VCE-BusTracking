## 1. Firebase Configuration (Android)

Copy the template and fill in your real values:
  android/app/google-services.json.example  →  android/app/google-services.json

Get values from: Firebase Console → Project Settings → Android App

---

## 2. Firebase Configuration (Dart / Multi-platform)

Copy the template and fill in your real values:
  lib/firebase_options.dart.example  →  lib/firebase_options.dart

OR regenerate using the FlutterFire CLI:
  dart pub global activate flutterfire_cli
  flutterfire configure

---

## 3. Google Maps API Key (Android & Web)

In **android/app/src/main/AndroidManifest.xml**, replace:
  android:value="YOUR_GOOGLE_MAPS_API_KEY"
...with your actual key from Google Cloud Console.

In **assets/map.html**, replace:
  https://maps.googleapis.com/maps/api/js?key=YOUR_GOOGLE_MAPS_API_KEY
...with your actual key.

IMPORTANT: After adding the real key locally, run these commands so Git ignores your local edits:
  git update-index --skip-worktree android/app/src/main/AndroidManifest.xml
  git update-index --skip-worktree assets/map.html
(This safely keeps the real key active on your machine, but prevents git from seeing it.)

---

## 4. Firebase Configuration (firebase.json)

This file is generated by the Firebase CLI. Regenerate it locally:
  firebase init

---

## Files that are GITIGNORED (you must create locally):
  - android/app/google-services.json         ← Firebase Android config
  - lib/firebase_options.dart                ← Firebase Dart config (multi-platform)
  - android/local.properties                 ← Local Android SDK path (auto-generated)
  - firebase.json                            ← Firebase CLI project config
  - secrets.properties                       ← (Optional) For injecting secrets via Gradle

## Files that STAY in the repo (with placeholders):
  - android/app/src/main/AndroidManifest.xml ← Google Maps key replaced with placeholder
  - assets/map.html                          ← Google Maps key replaced with placeholder
